NAME

gmake-transistors-sym − Make gEDA Transitor Symbols

SYNOPSIS

gmake-transistors-sym symbol_file

SYNOPSIS

gmake-transistors-sym [OPTIONS]TEMPLATE[.sym]PARAMETERS[.spm]

DESCRIPTION

gmake-transistors-sym Make gEDA Transitor Symbols

gmake-transistors-sym is for ’batch’ production of transistor symbols for the geda-gaf project. The script requires a template symbol file and a parameter file. Parameters read from the file are used in the output symbol file name or used as replacement values for pre-set variables in the symbol template file.

QUICK

usage: [-h, --help] | [--version] | [options] template[.sym] parameters[.spm] example: gmake-transistors-sym npn-example-template.sym params-exported-gnumeric.spm<ENTER>.

PARAMETERS

Fields in the TAB or space delimited parameter file are read and interpreted as follows:

1. Part Number string to be substituted for PART_NUMBER in the symbol file 2. Voltage string is included in the file name and as part of the COMPONENT_VALUE 3. Features string also included in the file name and in the COMPONENT_VALUE. This could be, for example, Wattage or Ampacity 4. Package name, if the -p option is present on the command line then the 4th field interpreted as the package ID and included in the file name 5. FootPrint must be the next argument and is substituted for unknown in the template 6. Optional symbol file modifier, if present, the value is used as a second suffix before dash suffix. Normally this would be used to select the template with the correct pin -out, for example using ECB or BEC. The string is prefixed with a underscore composing the template file name. 7. Optional symbol file suffix, if this value is present in the parameter file, the string will be used between the template given on the commanline and the symbol file extension, in this case the name of the template given on the command line should only be the base file name. The string is prefixed with a dash composing the template file name. Note that if a modifier is not given then this parameter 6 is assumed to be the symbol file suffix

INFORMATION

Use of the following options displays information related to gmake-transistors-sym but does not create any symbols:
-h, --help

Show information about the advanced usage of this command.

-u, --usage

Displays examples on usage"

--version

Displays software version information"

DIRECTIVES

-v, --verbose

Include additional information during processing.

OPTIONS

-p, --package

The packge type is included in the paratmeter data

-t, --template

Include the template prefix and suffix in the output symbol name

-m, --modifier

Suffix the template modifier in the output symbol name, like _ecb
